# CNEOX Platform Release Notes
## Release Date: December 30, 2025

---

##  Release: Authentication System Update - User ID Only

This release updates both the login and password reset systems to require only User ID for authentication, removing support for email and phone number login options. This change simplifies the authentication process, ensures consistent user identification, and resolves the issue where multiple accounts could share the same email address.

### Release Highlights
- ✅ Login page now only accepts User ID (CNEOX-XXXXXX format)
- ✅ Removed email and phone number login options
- ✅ Updated backend to require User ID only
- ✅ Simplified authentication flow
- ✅ Forgot password page now uses User ID instead of email
- ✅ Updated forgot password UI to match login page theme
- ✅ Backend forgot password endpoint updated to accept User ID
- ✅ Fixed text color visibility issue on dark-themed pages
- ✅ Removed global CSS override that forced black text on inputs
- ✅ Input fields now display white text on dark backgrounds
- ✅ **User ID format changed from CROWN-XXXXXX to CNEOX-XXXXXX**
- ✅ All existing database user IDs migrated to new format
- ✅ Migration script created for database updates
- ✅ All code references updated to use CNEOX format
- ✅ **Wallet display order rearranged on dashboard and withdrawal pages**
- ✅ **CNEOX logo in sidebar now links to homepage**
- ✅ **Logged-in users can now visit the landing page**
- ✅ **Removed all CROWN-XXX text references from website and code**
- ✅ **Navigation header now appears on all pages consistently**
- ✅ **Login and signup page logos now route to homepage**
- ✅ **All informational pages unlocked - logged-in users can access them**
- ✅ **Facebook and WhatsApp social media links added to landing page footer**
- ✅ **Fixed package creation/activation timeout and session expiration issues**

- ✅ Updated admin user ID from CROWN-000000 to CNEOX-000000 (with backward compatibility)

**Server-Side Updates**:
- `server/src/services/userId.service.ts` - Updated generation logic
- `server/src/controllers/auth.controller.ts` - Updated referrer validation
- `server/src/services/userInit.service.ts` - Updated admin checks

**Client-Side Updates**:
- All login/signup/forgot password pages - Updated placeholders
- All admin pages - Updated admin user checks
- All dashboard pages - Updated admin detection
- All informational pages - Updated example user IDs

**Migration Script**:
- Created `server/src/scripts/migrateUserIdCrownToCneox.ts`
- Migrates all existing CROWN-XXXXXX user IDs to CNEOX-XXXXXX
- Includes verification and error handling
- Safe to run multiple times (skips already migrated users)

**Before**:
```typescript
// User ID generation
return `CROWN-${formattedNumber}`;

// Admin check
if (user.userId === "CROWN-000000") { ... }

// Placeholder text
placeholder="User ID (CROWN-XXXXXX)"
```

**After**:
```typescript
// User ID generation
return `CNEOX-${formattedNumber}`;

// Admin check (with backward compatibility)
if (user.userId === "CNEOX-000000" || user.userId === "CROWN-000000") { ... }

// Placeholder text
placeholder="User ID (CNEOX-XXXXXX)"
```

**Migration Instructions**:
1. Backup your database before running the migration
2. Run the migration script: `npx ts-node -r dotenv/config src/scripts/migrateUserIdCrownToCneox.ts`
3. Verify all users have been migrated
4. The script will show a summary of migrated users
